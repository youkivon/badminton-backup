# 羽毛球视频分析小程序 - 产品规划

## 核心价值

看清自己的问题 + 进步对比

**设计原则：** 每次分析 = 给球员建一份技术档案（可积累，非一次性）

**产品定位：** 不是替代教练，是教练的鉴定工具——帮客户解决"我练了半天，到底有没有进步"这个不确定性。定价对标线下教练课，而非视频网站。

---

## 一、技术档案系统（持久化存储）

每位球员一份独立档案，记录历次分析结果。

**目录结构：**
```
~/BadmintonPlayers/
├── zhangsan/
│   ├── profile.json        # 球员基本信息、打法标签、技术问题标签
│   ├── history/
│   │   ├── 2026-05-01.json  # 第1次分析结果（含评分）
│   │   └── 2026-05-05.json  # 第2次分析结果（含评分）
│   └── frame_archive/       # 历次分析帧图存档
└── lisi/
    └── ...
```

**profile.json 内容：**
- 球员姓名/昵称、常用打法（双打前场/单打后场等）
- 技术标签：["握拍过紧", "重心过高", "步法跟不上"]
- 首次分析日期、最近分析日期
- 历次各动作评分（每次分析累加，形成评分曲线）

## 二、评分体系

增加客户粘性的核心机制。进步看得见，粘性自然来。

### 2.1 单次评分

- 每个动作独立打分（发力、步伐、闪腕、髋-肩分离、持拍手等）
- 单项分：0-10分（0=完全错误，10=标准动作）
- 整体技术分：各项动作加权平均，100分制或10分制
- 本次分析完立即输出："你的羽毛球技术总分：7.2分"

### 2.2 进步对比（纵向）

- 同一种错误在本次 vs 上次中的表现变化
- 判断逻辑：本次该错误帧数/总帧数 vs 历史累计频率
- 输出：「加重」「改善」「新出现」「已消除」
- 同动作帧图并排对比（上次 vs 本次）
- 总分变化曲线（生成曲线图）

### 2.3 同行横向对比（同难度组）

- 不跟职业选手比，跟同球龄、同水平的业余玩家比
- "你的反手发力已超过74%的同龄球友"
- 适度竞争心理激发练习动力
- 注意：新客户前3次不开放，避免当场劝退

## 三、客户粘性机制

核心逻辑：让客户感受到"每次分析都有新收获"，进步越快粘性越强。

### 3.1 成就系统（里程碑）

- 连续分析3次/7次/30次
- 某项动作首次突破8分
- 总分相比首次分析提升20%
- 达标即解锁成就徽章，有收集感

### 3.2 连续打卡 streak

- 每次分析打卡一次，连续7天、30天有不同奖励
- 断了会心痛（损失厌恶心理）
- 配合微信订阅消息提醒："您已连续分析X天，今天来打卡吗"

### 3.3 每月进步报告（可分享）

- 每月自动生成一份"这个月的进步报告"
- 配上前后期帧图对比 + 分數变化曲线
- 可以分享到朋友圈，是天然的获客内容
- 提供"一键生成海报"功能，含二维码

### 3.4 进步悬赏

- 系统自动指出"这个动作上次的得分最低，重点练一下"
- 客户练了两周回来分析，如果这个动作分数确实提高，主动给优惠券/加赠一次分析
- 把"主动练习"的动力形成闭环

### 3.5 社区挑战赛

- 每月发起一个技术挑战（如"这周练闪腕"）
- 参与者发视频，AI免费分析一次
- 挑战赛制有话题性，愿意主动传播
- 排行榜激励：进步最大的参与者获得奖励

## 四、报告类型选择（客户可选）

> 客户上传视频后，可自主选择报告类型：只出**战术报告**、只出**技术报告**、或者**两份都要**。战术和技术分析各自独立生成 PDF，可以单独购买，也可以打包购买后一起推送。

### 4.1 战术报告（Tactical Report）

面向：关注场上决策、跑位、配合的球员，尤其是双打爱好者。

**分析内容：**
- 球员站位判断：发球/接发/进攻/防守站位是否合理
- 双打轮转：轮转时机是否正确（何时该网前跟进、何时该后撤）
- 战术选择：放网前/抽档/挑后场/杀球等选择是否合理
- 攻守转换：被攻转守时的补位意识
- 搭档配合：两人之间的站位间距和分工

**输出特点：**
- 无技术动作诊断（如发力、闪腕等）
- 以战术选择合理性为主线
- 帧图标注站位和球路

### 4.2 技术报告（Technical Report）

面向：想专项提升动作质量的球员，参照本节第五节技术监理标准。

**分析内容：**
- 挥拍轨迹与击球点
- 发力链完整度（蹬地→转胯→转肩→挥拍）
- 闪腕时机与角度
- 步伐节奏与重心切换
- 各单项动作评分（参照2.1节）

**输出特点：**
- 有明确击球瞬间帧才输出技术诊断
- 准备/站位/过渡帧仅保留描述，不输出技术问题
- 不确定帧（含"疑似"/"？"等标记）直接跳过

### 4.3 综合报告（Full Report）

战术报告 + 技术报告全部输出，满足完整分析需求。

### 4.4 单独购买与打包

| 组合 | 内容 | 定价参考 |
|------|------|---------|
| 战术报告（单） | 站位/轮转/战术选择 | ¥9.9 |
| 技术报告（单） | 动作质量/发力/步伐 | ¥9.9 |
| 综合报告（打包） | 战术 + 技术两份 PDF | ¥15.9（略低于两份相加） |

> 打包定价逻辑：两份报告实际成本几乎为零（分析共用同一份视频帧图），适当让利促进全选。

## 五、技术监理标准（VLM 分析质量控制）

> 以下原则已通过实际数据验证（2026-05-10），是确保分析结果准确可靠的底线。

### 5.1 核心原则：宁可漏判，不能误判

**判断顺序：**
1. 球不可见 → 跳过
2. 挥拍轨迹不清晰 → 跳过
3. 逻辑矛盾（如标注"击球前瞬间"但输出完整发力诊断）→ 跳过

### 5.2 帧过滤规则（两层自动化过滤）

**第一层 — 不确定性跳过：**
`action_type` 含以下关键词 → 该帧不进入报告
- "未知"
- "疑似" / "**" / "？"（含任意不确定性标记）

**第二层 — 非击球帧技术诊断清空：**
`action_type` 含以下关键词 → 清空 errors/suggestions，技术评分置 0
- "非击球"
- "准备" / "站位" / "过渡"
- "等待" / "捡球" / "死球"

> **效果验证：** 应用两层过滤后，非击球帧不再输出"肘部发力"/"转体幅度"等仅适用于真实击球的技术诊断，报告准确性显著提升。

### 5.3 一致性校验层（商业化必需）

每次分析结束后自动执行，监控数据异常：

**校验指标：**
| 指标 | 说明 | 异常阈值 |
|------|------|---------|
| 跳帧率 | (跳过帧+清空帧) / 总分析帧 | >80% 触发预警 |
| 评分异常低 | ≤ 2 分的帧 | 触发人工复核标记 |
| 评分异常高 | ≥ 9 分的帧 | 触发完美性确认 |
| 逻辑矛盾 | 评分≥7 + 动作类型为"非击球/站位"等 | 自动清空诊断 |

**输出：** 控制台实时预警 + JSON 校验日志存档（`/audit/audit_日期时间.json`）

### 5.4 VLM 版本说明

- **当前有效版本：V1**（同数据集有效击球 5 帧，V2 因 Stage1 过严导致有效击球 0 帧）
- `run_analysis.py` 已强制锁定 V1，`use_v2` 逻辑已注释
- 如需重新启用 V2，必须先在同数据集上对比验证

### 5.5 PDF 中文渲染方案（已修复，2026-05-12）

- **方案**：fpdf2 + Unicode 中文字体（stheiti.ttf 华文细黑，已确认为 TrueType Unicode）
- **字体加载**：`add_font("STHeiti", fname=ASSETS_FONT, uni=True)`
- **图片宽高比**：动态计算（宽度优先，高度自适应，避免硬编码高度导致变形）
- **卡片高度**：文字超长时图片区域自动扩展，图文不重叠
- **验证方法**：生成的 PDF 用微信小程序打开，中文无乱码，图片比例正常

## 六、分析流程

1. 步骤1：视频上传 → 视频质检（分辨率/时长/帧率）
2. 步骤2：客户选择报告类型（战术/技术/综合，可多选）
3. 步骤3：选择/新建球员档案
4. 步骤4：帧提取 → VLM 动作分析（共一份视频帧图，战术和技术分析共用）
5. 步骤5：按报告类型组装内容，战术报告由战术 prompt 输出，技术报告由技术 prompt 输出，分别生成独立 PDF
6. 步骤6：若两项都选，打包推送两份 PDF
7. 步骤7：保存到该球员的历史档案（评分累加进历史记录）

**关键：** 分析结果必须持久化，不只是生成一次PDF就丢掉。

## 七、商业化四步流程

### 步骤1 — 视频质检（自动）

- 检查分辨率（≥720p）、时长（≤10分钟）、帧率（≥25fps）
- 检查画面稳定性和场地完整性
- 不合格：提示用户重新拍摄；合格：进入步骤2

### 步骤2 — 球员识别（新客户）

- 从视频关键帧提取所有球员人脸
- 客户上传自己的照片建立档案（已有档案则跳过）
- 列出场上所有球员供选择，建立客户档案（人脸+姓名+球龄等信息）

### 步骤3 — 关键动作分析（VLM）

- 帧差法抽帧 → 活跃帧筛选 → 针对选定球员逐帧技术分析
- 教练视角：发力链、步伐、持拍手、髋-肩分离、闪腕等
- 全程云端处理，Qwen-VL-Plus ¥0.0015/千tokens

### 步骤4 — 生成报告

- PDF/H5报告：动作分类、技术评分（各单项+总分）、改进建议、关键帧图例
- 对比历史档案：进步/退步项目标注
- 推送微信消息通知
- 解锁对应成就徽章（如有）

## 十四、质量控制检查清单

每次分析流程必须通过的检查项：

- [ ] `action_type` 含"未知/疑似/？/\*\*/" → 该帧跳过，不进入报告
- [ ] `action_type` 含"准备/站位/过渡/等待/捡球/死球" → 清空 errors/suggestions，quality=0
- [ ] 评分 ≥ 7 + 动作类型为"非击球/站位/过渡" → 自动清空并置 quality=0（矛盾检测）
- [ ] VLM 版本：确认使用 V1（V2 需重新验证后方可启用）
- [ ] PDF 生成：确认使用 fpdf2 + stheiti.ttf（uni=True）方案，中文字体加载正确
- [ ] 战术报告：无技术诊断内容；技术报告：只保留真实击球帧
- [ ] 一致性校验：跳帧率 >80% 触发预警，≤2分/≥9分触发复核标记
- [ ] 校验日志已写入 `players/[球员名]/audit/audit_日期时间.json`
- [ ] 图片宽高比：生成的 PDF 中图片不变形（宽度固定，高度按原始比例自适应）
- [ ] 图文重叠：shot 卡片内文字超长时不超出卡片区域
