# 羽毛球视频分析小程序 - 产品规划

## 核心价值

看清自己的问题 + 进步对比

**设计原则：** 每次分析 = 给球员建一份技术档案（可积累，非一次性）

**产品定位：** 不是替代教练，是教练的鉴定工具——帮客户解决"我练了半天，到底有没有进步"这个不确定性。定价对标线下教练课，而非视频网站。

---

## 一、技术档案系统（持久化存储）

每位球员一份独立档案，记录历次分析结果。

**目录结构：**
```
~/BadmintonPlayers/
├── zhangsan/
│   ├── profile.json        # 球员基本信息、打法标签、技术问题标签
│   ├── history/
│   │   ├── 2026-05-01.json  # 第1次分析结果（含评分）
│   │   └── 2026-05-05.json  # 第2次分析结果（含评分）
│   └── frame_archive/       # 历次分析帧图存档
└── lisi/
    └── ...
```

**profile.json 内容：**
- 球员姓名/昵称、常用打法（双打前场/单打后场等）
- 技术标签：["握拍过紧", "重心过高", "步法跟不上"]
- 首次分析日期、最近分析日期
- 历次各动作评分（每次分析累加，形成评分曲线）

---

## 二、评分体系

增加客户粘性的核心机制。进步看得见，粘性自然来。

### 2.1 单次评分

- 每个动作独立打分（发力、步伐、闪腕、髋-肩分离、持拍手等）
- 单项分：0-10分（0=完全错误，10=标准动作）
- 整体技术分：各项动作加权平均，100分制或10分制
- 本次分析完立即输出："你的羽毛球技术总分：7.2分"

### 2.2 进步对比（纵向）

- 同一种错误在本次 vs 上次中的表现变化
- 判断逻辑：本次该错误帧数/总帧数 vs 历史累计频率
- 输出：「加重」「改善」「新出现」「已消除」
- 同动作帧图并排对比（上次 vs 本次）
- 总分变化曲线（生成曲线图）

### 2.3 同行横向对比（同难度组）

- 不跟职业选手比，跟同球龄、同水平的业余玩家比
- "你的反手发力已超过74%的同龄球友"
- 适度竞争心理激发练习动力
- 注意：新客户前3次不开放，避免当场劝退

## 三、客户粘性机制

核心逻辑：让客户感受到"每次分析都有新收获"，进步越快粘性越强。

### 3.1 成就系统（里程碑）

- 连续分析3次/7次/30次
- 某项动作首次突破8分
- 总分相比首次分析提升20%
- 达标即解锁成就徽章，有收集感

### 3.2 连续打卡 streak

- 每次分析打卡一次，连续7天、30天有不同奖励
- 断了会心痛（损失厌恶心理）
- 配合微信订阅消息提醒："您已连续分析X天，今天来打卡吗"

### 3.3 每月进步报告（可分享）

- 每月自动生成一份"这个月的进步报告"
- 配上前后期帧图对比 + 分數变化曲线
- 可以分享到朋友圈，是天然的获客内容
- 提供"一键生成海报"功能，含二维码

### 3.4 进步悬赏

- 系统自动指出"这个动作上次的得分最低，重点练一下"
- 客户练了两周回来分析，如果这个动作分数确实提高，主动给优惠券/加赠一次分析
- 把"主动练习"的动力形成闭环

### 3.5 社区挑战赛

- 每月发起一个技术挑战（如"这周练闪腕"）
- 参与者发视频，AI免费分析一次
- 挑战赛制有话题性，愿意主动传播
- 排行榜激励：进步最大的参与者获得奖励

---

## 四、报告设计

### Part 1 - 本次发现（独立完整）

- 本次分析的N个动作帧
- 每个问题：错误类型 + 帧图证据 + 改进建议 + 本次评分
- 整体评分 + 各单项动作评分

### Part 2 - 进步对比（纵向）

- 核心问题：改善 / 加重 / 新出现 / 已消除
- 对比示例：上次「握拍过紧」(5分) → 这次「握拍有改善」(7分)
- 总分变化：6.5分 → 7.2分（+10.8%）
- 首次分析时这部分为空

## 五、分析流程

1. 步骤1：视频上传
2. 步骤2：选择/新建球员档案
3. 步骤3：帧提取 → VLM动作分析（发现N个技术问题 + 各动作评分）
4. 步骤4：保存到该球员的历史档案（评分累加进历史记录）
5. 步骤5：生成报告（含本次 + 对比 + 成就解锁）

**关键：** 分析结果必须持久化，不只是生成一次PDF就丢掉。

## 六、商业化四步流程

### 步骤1 — 视频质检（自动）

- 检查分辨率（≥720p）、时长（≤10分钟）、帧率（≥25fps）
- 检查画面稳定性和场地完整性
- 不合格：提示用户重新拍摄；合格：进入步骤2

### 步骤2 — 球员识别（新客户）

- 从视频关键帧提取所有球员人脸
- 客户上传自己的照片建立档案（已有档案则跳过）
- 列出场上所有球员供选择，建立客户档案（人脸+姓名+球龄等信息）

### 步骤3 — 关键动作分析（VLM）

- 帧差法抽帧 → 活跃帧筛选 → 针对选定球员逐帧技术分析
- 教练视角：发力链、步伐、持拍手、髋-肩分离、闪腕等
- 全程云端处理，Qwen-VL-Plus ¥0.0015/千tokens

### 步骤4 — 生成报告

- PDF/H5报告：动作分类、技术评分（各单项+总分）、改进建议、关键帧图例
- 对比历史档案：进步/退步项目标注
- 推送微信消息通知
- 解锁对应成就徽章（如有）

## 八、定价体系

**客户画像：** 大学毕业、70岁以内的羽毛球爱好者，可能是刚参加工作的年轻人到退休前的中老年人。时间相对灵活，但羽毛球本身耗体力，一周3-4次已是高频，一个月8-16次是正常活跃水平。

**定价逻辑：** 不以"每天"为单位，而是匹配"按次分析"的消费习惯。参考同类健身/运动类小程序主流定价（¥9.9-39区间），定位工薪族和学生都负担得起的价格带。

### 8.1 套餐设计

| 套餐 | 价格 | 说明 |
|------|------|------|
| 单次 | ¥9.9 | 零门槛体验，先试再买 |
| 4次包 | ¥29 | 平均¥7.25，匹配一个月4-8次的活跃频率 |
| 12次季度包 | ¥79 | 平均¥6.6，对应一季度约12-16次打球 |
| 年卡 | ¥199/年 | 锁定年度用户，每天不到6毛钱 |

**说明：** 不设"月卡"（容易让人误解为一个月内无限次），改为"4次包"更符合按次消费的心理预期。

### 8.2 定价锚点话术

- 季度¥99 = 一桶球线 + 一场球馆费，但能让你知道自己两个月有没有白练
- 年卡¥199 = 每天不到6毛钱，相当于一杯奶茶钱
- 对比教练课：一节私教¥100~300，我们一个月¥39，相当于一节教练课的零头

### 8.3 年卡后续规划

- 第一阶段（先跑通）：单次 + 月卡 + 季度卡
- 第二阶段（有用户基础后）：推出年卡¥199/年
- 年卡专属权益：每月进步报告、成就徽章体系、优先社区挑战赛、进步悬赏奖励翻倍

### 9.4 关键洞察

- 云端成本（VL+人脸）占总成本比例极小，规模效应主要体现在固定成本摊薄
- 微信支付手续费（营收×0.6%）是随规模增长的主要变动成本
- 服务器¥200/月是最大可控固定成本，100客户 vs 10000客户成本不变
- 100客户阶段：月均利润¥689，目标是积累口碑而非盈利
- 1000客户阶段：月均利润¥14,547，可以开始考虑招人
- 10000客户阶段：月均利润¥201,126，是稳定盈利的业务
- 本表未含：营销推广、人工客服、产品迭代开发成本（实际需预留营收20~30%）
- **频率修正：** 羽毛球爱好者每月4-8次是正常活跃水平，因此4次包/季度包消耗周期较长，年卡用户复购意愿更强

### 9.5 盈亏平衡点

- 最低盈亏（月）：固定成本¥200 ÷ 净利润率76.7% ≈ ¥261/月营收
- 即每月只要有约26个客户买1次单次，就能覆盖服务器成本
- 实际上只要维持30~50个活跃用户（每月至少1次分析），业务就不会亏损
